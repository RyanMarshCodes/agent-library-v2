# Execution Policy (Universal)

## Objective
Deliver correct, maintainable changes with minimal overhead and clear validation.

## Policy

1. Start with a short explicit plan.
2. Keep scope tight to requested outcomes.
3. Prefer smallest viable change set.
4. Validate behavior before considering task done.
5. Surface assumptions and blockers quickly.

## Required Sequence

1. Gather context.
2. Plan in ordered steps.
3. Implement in focused batches.
4. Validate changed behavior.
5. Report results and risks.
