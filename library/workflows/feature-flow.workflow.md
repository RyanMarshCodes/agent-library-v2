# Workflow: Feature Flow

## Purpose
Implement a feature from request to verified change.

## Required Inputs

1. Feature statement.
2. Acceptance criteria.
3. Scope boundaries and constraints.

## Steps

1. Build context pack.
2. Produce implementation plan.
3. Implement in focused change batches.
4. Run targeted validation.
5. Run review gate.
6. Publish outcome summary.

## Required Outputs

1. Change summary.
2. Validation evidence.
3. Residual risks and follow-ups.
