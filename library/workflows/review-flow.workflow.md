# Workflow: Review Flow

## Purpose
Assess implementation quality before merge or release.

## Required Inputs

1. Diff summary.
2. Acceptance criteria.
3. Validation results.

## Steps

1. Check correctness against acceptance criteria.
2. Check regression and edge-case risk.
3. Check security/performance implications.
4. Check test and validation coverage.
5. Publish findings ordered by severity.

## Required Outputs

1. Findings list with severity and evidence.
2. Open questions or assumptions.
3. Merge recommendation.
