# Workflow: Release Flow

## Purpose
Prepare and validate a change set for safe release.

## Required Inputs

1. Release scope.
2. Change summary.
3. Rollback strategy.

## Steps

1. Confirm release scope and dependency readiness.
2. Verify build/test status for release candidates.
3. Confirm migration and config readiness.
4. Prepare release notes and risk log.
5. Confirm rollback instructions.
6. Publish release checklist result.

## Required Outputs

1. Release readiness decision.
2. Risk and rollback notes.
3. Post-release verification checklist.
